  <action>
    1. Install dependencies:
       ```bash
       npm install sonner @heroicons/react clsx tailwind-merge
       ```

    2. Update `tailwind.config.ts` — extend theme with:
       - `colors.burgundy` scale (50-950) using these values from research:
         50: '#fdf4f5', 100: '#fbe8eb', 200: '#f8d5db', 300: '#f2b3bd',
         400: '#e98799', 500: '#db5c77', 600: '#c53d5c', 700: '#a6294b',
         800: '#8a2442', 900: '#76223d', 950: '#420e1e'
       - `colors.cream` scale: 50: '#fdfaf7', 100: '#faf3e8', 200: '#f5ead6'
       - `boxShadow`: 'soft': '0 2px 8px 0 rgb(0 0 0 / 0.06)', 'card': '0 4px 12px 0 rgb(0 0 0 / 0.08)', 'card-hover': '0 8px 20px 0 rgb(0 0 0 / 0.12)'
       - Keep existing `colors.background` and `colors.foreground` CSS variables

    3. Update `src/app/globals.css`:
       - Change `:root` `--background` from `#ffffff` to `#fdfaf7` (warm cream)
       - Change `:root` `--foreground` from `#171717` to `#420e1e` (deep burgundy 950)
       - Remove the `prefers-color-scheme: dark` media query (no dark mode in MVP)
       - Keep font-family as-is (system fonts for performance)

    4. Create `src/lib/utils.ts`:
       - Export `cn()` function using clsx + tailwind-merge:
         ```ts
         import { clsx, type ClassValue } from "clsx";
         import { twMerge } from "tailwind-merge";
         export function cn(...inputs: ClassValue[]) { return twMerge(clsx(inputs)); }
         ```
  </action>
  <verify>
    - `npm run build` passes without errors
    - `tailwind.config.ts` contains `burgundy` color object with keys 50-950
    - `globals.css` has `--background: #fdfaf7`
    - `src/lib/utils.ts` exports `cn` function
  </verify>
  <done>Tailwind config has burgundy scale, cream colors, custom shadows. CSS variables use warm cream background. cn() utility available.</done>
