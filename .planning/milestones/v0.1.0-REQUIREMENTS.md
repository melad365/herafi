# Requirements: Herafi

**Defined:** 2026-02-07
**Core Value:** Anyone can find and hire a trusted service provider for in-person or digital work through a simple, browsable marketplace with clear service listings.

## v1 Requirements

### Authentication

- [x] **AUTH-01**: User can sign up with email and password — ✅ v0.1.0
- [x] **AUTH-02**: User can log in with email and password — ✅ v0.1.0
- [x] **AUTH-03**: User session persists across browser refresh — ✅ v0.1.0

### Profiles

- [x] **PROF-01**: User can set display name, avatar, and bio — ✅ v0.1.0
- [x] **PROF-02**: User profile shows services offered and aggregate rating (provider view) — ✅ v0.1.0
- [x] **PROF-03**: User can upload portfolio/work sample images — ✅ v0.1.0

### Service Listings

- [x] **GIG-01**: Provider can create a gig with title, description, and category — ✅ v0.1.0
- [x] **GIG-02**: Provider can set pricing tiers (Basic/Standard/Premium) per gig — ✅ v0.1.0
- [x] **GIG-03**: Provider can edit and delete their gigs — ✅ v0.1.0
- [x] **GIG-04**: Gig detail page shows full description, images, tiers, provider info, and reviews — ✅ v0.1.0

### Search & Discovery

- [x] **DISC-01**: User can browse services by category (Plumbing, Painting, Cleaning, Carpentry, Welding, Digital, etc.) — ✅ v0.1.0

- [x] **DISC-03**: User can filter results by price range and rating — ✅ v0.1.0

### Orders & Payments

- [x] **ORD-01**: User can place an order by selecting a gig tier — ✅ v0.1.0
- [x] **ORD-02**: Mock payment flow auto-approves on user confirmation — ✅ v0.1.0

### Messaging

- [x] **MSG-01**: User can send and receive real-time chat messages with another user — ✅ v0.1.0

### Reviews

- [x] **REV-01**: User can leave a star rating and written review — ✅ v0.1.0

### Provider Dashboard

- [x] **DASH-01**: Provider can manage their gigs from a dashboard — ✅ v0.1.0
- [x] **DASH-02**: Provider can view incoming orders — ✅ v0.1.0
- [x] **DASH-03**: Provider can access their message conversations — ✅ v0.1.0

## Out of Scope

| Feature | Reason |
|---------|--------|
| Location/map-based search | Deferred — not needed for MVP, adds significant complexity |
| Real payment processing / escrow | Mock payments only — validate demand before Stripe complexity |
| Mobile native app | Web-first — responsive design sufficient for MVP |
| Video content in listings | Storage/bandwidth costs — defer to v2+ |
| Dispute resolution system | No real payments — no disputes to resolve yet |
| Multi-language support | Single market first |
| Background checks for providers | Regulatory complexity — defer to v2+ |
| Service subscriptions (recurring bookings) | Different business model — defer |
| API for third-party integrations | No ecosystem yet |
| Bidding/auction system | Race-to-bottom pricing — use fixed tiers instead |
